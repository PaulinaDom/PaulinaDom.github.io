Power BI · DAX · Star Schema · Project 04

Sales Performance
YTD vs PYTD Dashboard

Power BI DAX Star Schema Time Intelligence Dynamic Switching

A sales analytics dashboard focused on Year-to-Date vs Prior Year-to-Date comparison, allowing dynamic switching between Sales, Quantity, and Gross Profit across time, country, customer, and product dimensions.

⚠ Based on a tutorial by Mo Chen (GitHub / YouTube). My own implementation — with focus on understanding and documenting each modelling and DAX decision from first principles.
Data Architecture

Star schema design

The model follows Power BI best practices: a central fact table with one-to-many relationships to dimension tables, single-direction filtering, and a dedicated measures table for organization.

⬡
Fact_Sales
Central transactional table with Product_id, Account_id, Date, Sales_USD, Quantity, COGS_USD. 2022–2024 data.
◻
Dim_Date
Custom calendar table built in DAX. Includes InPast calculated column for correct PYTD comparisons.
◻
Dim_Customer
Account-level descriptive context, linked via Account_id.
◻
Dim_Product
Product hierarchy: type and name for drill-down in waterfall charts.
◻
Dim_Country
Geographical dimension enabling country-level performance analysis.
⊘
Slicer_Values
Disconnected table (Sales / Quantity / Gross Profit) powering dynamic metric switching across all visuals.
DAX Design

Time intelligence & dynamic measures

InPast — enabling correct PYTD

Calculated column on Dim_DateDAX
-- Identifies dates eligible for prior-year comparison
Inpast =
VAR lastsalesdate = MAX(Fact_Sales[Date])
VAR lastsalesdatePY = EDATE(lastsalesdate, -12)
RETURN
    Dim_Date[Date] <= lastsalesdatePY

YTD & PYTD measures

Year-to-date and Prior Year-to-date — SalesDAX
YTD_Sales = TOTALYTD([Sales], Fact_Sales[Date])

PYTD_Sales =
CALCULATE(
    [Sales],
    SAMEPERIODLASTYEAR(Dim_Date[Date]),
    Dim_Date[Inpast] = TRUE
)

Dynamic switching — one slicer controls all visuals

Switch measures using SELECTEDVALUE() + SWITCH()DAX
Switch_YTD =
VAR selected_value = SELECTEDVALUE(Slicer_values[Values])
VAR result =
    SWITCH(selected_value,
        "Sales",        [YTD_Sales],
        "Quantity",     [YTD_Quantity],
        "Gross Profit", [YTD_GrossProfit],
        BLANK()
    )
RETURN result

-- Difference measure for KPI cards and waterfall chart
YTD vs PYTD = [Switch_YTD] - [Switch_PYTD]
Visual Design

5 chart types, all dynamic

KPI Cards
YTD, PYTD, YTD vs PYTD difference, and % Gross Profit. Conditional formatting: red background for negative variance, green for positive.
Column & Line Chart
X-axis: month from Dim_Date. Columns: Switch_YTD, Line: Switch_PYTD. Enables trend comparison across the selected year and metric.
Scatter Plot
X: Switch_YTD, Y: % Gross Profit, Values: Account. Average reference lines on both axes. Zoom enabled. Identifies which accounts drive volume vs margin.
Waterfall Chart
Y: YTD vs PYTD. Category hierarchy: Month → Country → Product Type → Product Name. Full drill-down from time period to individual product.
Treemap
Country by YTD vs PYTD. Bottom-N filter shows 10 worst-performing countries. Cross-filters with all other visuals for integrated exploration.

All visual titles and the report title use DAX SELECTEDVALUE() measures — so every title dynamically reflects whatever metric and year is currently selected.
